# Contributing to millfolio

Thanks for your interest! A note on how this project takes contributions.

## Bug reports, ideas, and questions — always welcome
Open an **issue** or a **GitHub Discussion**. Clear reproductions, logs, and
version info help a lot. For a suspected **security or privacy issue**, please use
GitHub's private "Report a vulnerability" (the Security tab) or contact the
maintainer directly — not a public issue.

## Code changes — please open an issue first
millfolio is an early, security-sensitive project (it handles people's financial
documents), so code is reviewed carefully:

- **Open an issue to discuss before sending a pull request.** Unsolicited PRs —
  especially large or AI-generated ones — may be declined without review.
- Keep changes small and focused, and include tests where it makes sense.

## Contributor License Agreement (CLA)
millfolio may be offered commercially in the future, so the project must be able to
relicense if needed. **Every code contribution requires a signed CLA.** On your
first pull request, the CLA Assistant bot will ask you to sign with a one-time
comment. The agreement is here:
**https://github.com/millfolio/.github/blob/main/CLA.md**

By contributing you agree your contribution is licensed under the project's license
(**Apache-2.0**) and you grant the maintainer the rights described in the CLA.

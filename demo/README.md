# millfolio demo

> 💬 **Community:** questions, ideas, and show-and-tell live in [GitHub Discussions](https://github.com/millfolio/millfolio/discussions).

A **public, self-serve demo** of millfolio over a **synthetic vault** — *replayed
codegen, real execution*. The real millfolio code is reused **unmodified**; this
repo wires it up via configuration and a small replay proxy. The real code has no
knowledge of the demo.

## How it works — the replay seam

The real app server writes each vault program by POSTing to a **configurable**
endpoint (`{ANTHROPIC_BASE_URL}/messages`, Anthropic Messages API shape). The demo
points `ANTHROPIC_BASE_URL` at the **replay proxy** (`replay/proxy.py`) instead of
`api.anthropic.com`:

```
visitor ─▶ app server (REAL, unmodified)
              │  aliased manifest + question
              ▼
          replay proxy  ──HIT──▶ cached program  (no paid API call)
              │  MISS
              ├─ capture mode → real Anthropic API → store program   (priming only)
              └─ replay  mode → friendly fallback program            (public-safe)
              ▼
          REAL sandbox: compile + run the program over the SYNTHETIC vault → answer
```

Replayed programs run **for real** over the synthetic vault, identical every time.
Programs that call `ask_local()`/`search()` use the on-device **inference engine**
(`:8001`, shared weights) — real local inference, still no paid API. The sandboxed
run is serialized by a **FIFO run-queue** and visitors see their position. See
[`RUNBOOK.md`](RUNBOOK.md) for the full setup (shared weights → engine on the main
account → app on bgent → prime → deploy).

**Nothing public ever reaches a paid API.** Codegen is replayed; misses get a
fallback program; only the local engine runs at execution time.

## Layout

```
replay/proxy.py     the codegen replay/capture proxy (stdlib python3)
replay/cache/       question → cached program (committed; THIS is the demo)
fixtures/           synthetic statements + their pre-built index (see fixtures/README.md)
scripts/run-demo.sh    launch proxy (replay) + the REAL app server, pointed at fixtures
scripts/prime-cache.sh fill the cache once, with a real key, from the REAL model
web/                optional demo landing / suggested-question chips
millfolio/          the real product as a git submodule (or consume the release bundle)
```

## Reusing the real code

The demo depends on the **built** millfolio app server + bundle, not a fork. Two
options (no real-code changes either way):
- **Submodule** the `millfolio/millfolio` superproject and build from source, or
- **Consume the released bundle** (`millfolio.zip`) — the unmodified product.

`run-demo.sh` points `MILLFOLIO_SERVER` / `MILLFOLIO_BUNDLE` at whichever you use.

## Run

```bash
# 1) build synthetic fixtures + index them (see fixtures/README.md)
# 2) prime the cache ONCE from the real model:
DEMO_CAPTURE_KEY=sk-ant-...  bash scripts/prime-cache.sh
# 3) serve the public demo (replay only, no paid API):
bash scripts/run-demo.sh        # → http://localhost:10010
```

## Synthetic data

The demo vault is the separate **`millfolio/demo-vault`** repo (fictional Riverbank
statements + insurance/registration, generated by `generate.py`). Add it as a
submodule at `fixtures/` (or point `DEMO_VAULT` at a sibling checkout) and index it
once — chunks + the reconciled `transactions.tsv` — into the demo account's
`~/.config/millfolio`.

## Hosting

Runs in a **dedicated macOS account** on the Mac mini (the per-query sandbox is
Seatbelt). Publish `:10010` at **`demo.millfolio.app`** via **Cloudflare Tunnel**
(`scripts/tunnel.sh` + `replay/cloudflared-config.example.yml`) — Cloudflare handles
DNS + TLS for the custom domain, it's outbound-only (no open ports), and visitors
need nothing. (Tailscale Funnel can't serve a custom domain — only `*.ts.net`.)
Safety: synthetic data only, the EgressGuard + sandbox still run, and replay mode
never calls a paid API. Add a Cloudflare rate-limit rule for the public endpoint.
